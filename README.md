# mundIAl — Tu tiempo, tu Mundial

FIFA World Cup 2026 match recommendation system.

## Stack

| Layer | Tech | Deploy |
|---|---|---|
| Frontend | Vite + React + TypeScript | Vercel |
| Backend | FastAPI + Python 3.12 | Render |
| Database | PostgreSQL 16 (port 5436 locally) | Render |
| Package mgr | uv | — |

---

## First-time setup

```bash
# 1. Copy env files
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env

# 2. Install frontend deps
cd frontend && npm install && cd ..

# 3. Start Postgres + backend containers
docker compose up --build -d

# 4. Run migrations
cd backend && uv run alembic upgrade head

# 5. Seed match + team data
uv run python -m app.seed.seed

# 6. Start frontend dev server (separate terminal)
cd frontend && npm run dev
```

| Service | URL |
|---|---|
| Frontend | http://localhost:5173 |
| Backend API | http://localhost:8000 |
| Swagger docs | http://localhost:8000/docs |
| Postgres | localhost:**5436** |

## Daily dev commands

### Backend

```bash
cd backend

# Start only the DB (if you want to run FastAPI outside Docker)
docker compose up db -d

# Run the API locally (hot reload)
uv run uvicorn app.main:app --reload

# Install a new dependency
uv add <package>

# Run all quality gates
uv run ruff check app/
uv run ruff format app/
uv run mypy app/
uv run pytest tests/

# Auto-fix lint issues
uv run ruff check app/ --fix
```

### Database / Alembic

```bash
cd backend

# Apply all pending migrations
uv run alembic upgrade head

# Create a new migration (after changing ORM models)
uv run alembic revision --autogenerate -m "describe_change"

# Roll back one migration
uv run alembic downgrade -1

# Show current migration state
uv run alembic current

# Re-seed data (idempotent — safe to run multiple times)
uv run python -m app.seed.seed
```

### Frontend

```bash
cd frontend

# Dev server
npm run dev

# Type check
npx tsc --noEmit

# Production build
npm run build

# Preview production build locally
npm run preview
```

### Docker

```bash
# Start all services in background
docker compose up -d

# Start only the DB
docker compose up db -d

# Rebuild backend image after Dockerfile or dep changes
docker compose up --build backend

# View logs
docker compose logs -f backend
docker compose logs -f db

# Stop everything
docker compose down

# Stop and wipe the DB volume (full reset)
docker compose down -v
```

## Monorepo structure

```
mundIAl/
├── frontend/          # Vite + React
├── backend/           # FastAPI
│   ├── app/
│   │   ├── core/      # config (pydantic-settings)
│   │   ├── db/        # session, ORM models (Team, Match)
│   │   ├── ml/        # feature engineering, classifier, explainer
│   │   ├── modules/   # recommendations (routes/service/repo/schemas)
│   │   ├── models/    # classifier.pkl (generated by notebook)
│   │   └── seed/      # teams.json, matches.json, seed.py
│   ├── alembic/       # migrations
│   └── notebooks/     # EDA + model training
└── docker-compose.yml
```

## API

```
POST /api/v1/recommend
{
  "favorite_teams": ["Argentina"],
  "favorite_players": ["Lionel Messi"],
  "available_slots": [{ "day_of_week": "saturday", "start_hour": 14, "end_hour": 23 }],
  "timezone": "America/Argentina/Buenos_Aires",
  "country": "AR"
}
```

Response groups all 72 first-round matches into `imperdible`, `vale_la_pena`, `para_el_resumen`
with a per-match `score`, `explanation`, and full `score_breakdown` (11 features).
