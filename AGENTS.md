# mundIAl вЂ” AGENTS.md

## Project Overview

Competition entry for **"Tu tiempo, tu Mundial"** (Universidad Austral, Facultad de IngenierГ­a).

**Goal:** Recommend which FIFA World Cup 2026 first-round matches a user should watch, given their profile (favorite teams, players, available time slots). Each of the 72 group-stage matches is classified into one of three categories:

- **Imperdible** вЂ” must-watch: high affinity + good slot
- **Vale la pena** вЂ” worth watching: interesting but not crucial, or awkward time
- **Para ver el resumen** вЂ” watch highlights: low affinity or impossible schedule

**Deadline:** June 4, 2026. Demo must be publicly accessible.

## Recommendation Engine

### Feature Engineering (`app/ml/feature_engineering.py`)

For each of the 72 matches, a feature vector is computed given the user profile.
Note: `end_hour=24` in availability slots is supported (handled as midnight next day).

| # | Feature | Description | Range |
|---|---|---|---|
| 0 | `team_affinity` | User's favorite team is playing | 0/1 |
| 1 | `rival_affinity` | Rival of user's favorite team is playing | 0вЂ“0.6 |
| 2 | `star_player_playing` | User's listed player is in a squad | 0/1 |
| 3 | `availability_score` | Fraction of 2-hr match window in user's free slots | 0вЂ“1 |
| 4 | `timezone_penalty` | Inconvenience of local kickoff hour (late night = 1) | 0вЂ“1 |
| 5 | `rivalry_index` | H2H historical rivalry intensity (pre-annotated per match) | 0вЂ“1 |
| 6 | `star_power` | Combined star-player quality of both squads | 0вЂ“1 |
| 7 | `group_stakes` | Group qualification importance by round (R1=0.55, R2=0.75, R3=0.95) | 0вЂ“1 |
| 8 | `expected_competitiveness` | Closeness of FIFA rankings (equal = 1) | 0вЂ“1 |
| 9 | `narrative_score` | Pre-annotated match significance (opener, classic rematch, etc.) | 0вЂ“1 |
| 10 | `regional_affinity` | User's country shares confederation with a team | 0/1 |

### ML Classifier (`app/ml/classifier.py`)

- **Model:** Random Forest (scikit-learn), trained in `notebooks/02_model_training.ipynb`
- **Input:** 11-dim feature vector per (user, match) pair
- **Output:** `imperdible` / `vale_la_pena` / `para_el_resumen`
- **Training data:** ~20 diverse user personas Г— 72 matches в‰€ 1,440 labeled rows, annotated by the team
- **Validation:** Stratified k-fold CV across personas; feature importance plot expected to show `team_affinity` and `rivalry_index` as top predictors
- **Fallback:** If `app/models/classifier.pkl` does not exist, a rule-based weighted-sum heuristic is used automatically (same 11 features, hardcoded weights). This keeps the API functional before the notebook is run.

### Explanation (`app/ml/explainer.py`)

Template-based in **Spanish**: identifies the top-scoring positive feature and the main obstacle (timezone/availability) to generate a one-sentence explanation per match. Post-MVP upgrade path: Codex API for natural language explanations.

## Data

All data is static and committed to the repo. No external API calls at inference time.

- **`teams.json`**: 48 WC 2026 teams with FIFA April 2026 ranking, confederation, rival team names, top 5 key players, star_power (0вЂ“10), narrative_flags (defending_champion, host_nation, etc.)
- **`matches.json`**: 72 group-stage matches (Groups AвЂ“L) with real UTC datetimes, venue, city, pre-annotated `narrative_score` (0вЂ“10) and `rivalry_index` (0вЂ“10)
- **`src/data/`**: Static TS fallback data (countries, teams, timezones, players) used by frontend before API responds or when offline (PWA)
- Data is seeded into PostgreSQL via `uv run python -m app.seed.seed` (idempotent)

## Frontend Architecture

- **Router:** TanStack Router (file-based, `autoCodeSplitting: true`). `routeTree.gen.ts` is auto-generated вЂ” never edit manually.
- **Data fetching:** TanStack Query (`staleTime: 5min`). Each data concern has a dedicated hook in `src/hooks/`.
- **UI:** shadcn/ui primitives in `src/components/ui/` + custom domain components in `src/components/`.
- **PWA:** `vite-plugin-pwa` with `generateSW` strategy. Icons in `public/`. Service worker precaches all static assets; `/api/v1/*` uses `NetworkFirst` (1hr cache). Installable on Vercel (HTTPS required вЂ” satisfied automatically).
- **Env var:** `VITE_API_URL` вЂ” defaults to `http://localhost:8000` if not set.

## Backend Conventions

Follows the project's backend skill (SOLID / RORO):

- **Routes** (`_routes.py`): HTTP contract only вЂ” delegates immediately to service
- **Service** (`_service.py`): RORO orchestration вЂ” receives typed input DTO, returns typed output DTO; no HTTP objects
- **Repository** (`_repository.py`): DB queries via `AsyncSession` вЂ” no business logic
- **ML modules** (`ml/`): Pure functions вЂ” no FastAPI or DB dependencies
- **Catalog module** (`modules/catalog/`): Shared read-only schemas and repository for teams/countries/timezones; imported by setup and players modules
- **Dependency injection** via `Annotated[AsyncSession, Depends(get_db)]`
- **Package manager:** `uv` вЂ” always use `uv run` prefix
- **Startup:** `lifespan` context manager loads `classifier.pkl` into memory once
- **SQL echo:** disabled (`echo=False`) in all environments
